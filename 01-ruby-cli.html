Writing your first Ruby CLI application

Introduction

Congratulations on taking your first steps toward learning ruby! In this tutorial we'll cover:

  • Creating a Ruby program and running it in Terminal
  • Making that program run in Terminal no matter what directory you're in
  • Reading from and writing to files in Ruby

We'll do this by creating a command line application that lets you manage a list of questions that you might have and don't want to forget. The application will let you display the list, and add and remove items. It wouldn't be much of a memory aid if the list disappeared every time the program stopped running, so we'll also make it save your list to a text file so it can access it whenever.

Getting Started

Hello World

First we'll cover the basics of creating and running a program in Ruby. One popular way of showing basic program structure for someone learning a new language is to show them a Hello World program. Hello World is a program that outputs the line of text "Hello World" to the console. Sounds simple, doesnt it? It is!

In Ruby, "Hello World" looks something like this: 

 puts "Hello World"

Pretty straightforward, right? Here puts is a Ruby command that tells your program to write the statement that follows to the Terminal window.

Go ahead and type or cut-and-paste that line of code into your text editor and save it. Call it hw.rb

Running your first Ruby program

Now that you have written your first program, let's learn how to run it and see what happens. In the terminal type ruby hw.rb and press enter. You should see the words "Hello World" appear on the next line of the terminal.

Congratulations! You just wrote and ran your first Ruby program

Make a program run anywhere

If our questions app is going to be useful, it should be easy to run. ruby hw.rb isn't too bad to type, but our program is eventually going to use command line options to add and remove questions. That could wind up looking something like: ruby questions.rb add "Why do I have to keep typing all this out every time?"

Wouldn't it be better if we could just type: questions add "This is nice, isn't it?"

The command to execute our new program was only one line, but there are a couple of things happening behind the scenes when you hit enter.

Tell your shell what to use to run your program

Right now we're typing ruby each time we run our program. That explicity tells our shell that we want to use the ruby interpreter to run our code. The ruby interpreter sees the .rb part of the file name and knows that this is a file it can run. If we're going to get the shell to figure things out on its own it's going to need a hint.

We can give the shell the hint it needs by adding the following to the beginning of our code: 

 	#!/usr/bin/env ruby

This line tells your shell to use ruby to run the code in this file, and where to find the ruby interpreter. That will let us skip typing ruby every time, and we can now rename our file to remove the .rb as well.

Tell your filesystem that it's okay to run the program

With that done you might think you're all set, but our file will need permission from the file system in order to run. New files in a unix-like file system don't have this permission, so we'll have to set it like this: 

 chmod u+x questions

This line tells your shell to modify the permissions for questions. The u+x part says to "+"add the e"x"ecute permision for "u"sers.

Tell your shell where to find your program

Whenever you try to execute a command in the shell it looks through a list of locations for the program name you entered.

Next steps in Ruby

Variables

In much the same way you learned in Algebra class how to use letters like "x" and "y" as placeholders for numbers in mathematical operations, Ruby lets us define variables to use as placeholders in our code. The biggest difference is that, with Ruby, variables can hold amost any kind of information.

You can create a new variable, or change the value of an existing one by writing some_variable = "some_value". In this case we created/assigned a variable named some_value and assigned it a String (most items in Ruby with quotation marks around them are Strings).

Data Types and Operations

What is a, "kind of information?" Take a second to think about it. It might help to ask yourself:

  • What do all words have in common?
  • All numbers?
  • What is the difference between a number and a word?

Don't worry. These are things we grow up having an innate sense for, so it can be hard to quickly come up with an explanation. There is probably more to it, but the important takeaways for our purposes are:

  1. Every kind of information or, "Data Type" in programmer-speak has a value
  2. A Data Type has a set of actions or, "Operations" help us do things we normally need to do to manipulate that value

It's probably starting to make some sense now, but an easy way to solidify things might be to take a look at some common Data Types and the Operations that Ruby (and most other languages) give us to use the information they store productively.

Data Type Stores Common Operations
String Words and characters make uppercase/lowercase
Integer Whole numbers Mathematical operations like +,-,*,/
Float Numbers with decimal points +, -, *, /, round
Array A list of things add/remove items

An individual example of a Data Type is called an, "Instance." In other words, the whole number 123

Functions

When we try to solve most tasks we can usually identify certain sets of steps that get repeated. Maybe we're unloading the dishwasher. For every item you unload you might pick it up, wipe any moisture off of it, and place it in the appropriate cupboard or cabinet. If you were writing a program to unload a dishwasher that holds 50 items you could type those three steps 50 times, or you could call the process something like, "unload an item" and cut your typing by 2/3rds.

That shortening process is like a programmer writing a function, and we can save ourselves a lot of work that way. Here's an example of a Ruby function: 

def add(newQuestion)
  @questionList.push(newQuestion)
  write_to_file
end

Don't worry if that looks confusing for now. We'll break it down line-by-line in a bit.

Loops

Functions are nice, but we want to be extra lazy. Loops let us do the equivalent of saying, "unload an item 50 times," most of the time Ruby can do even better and let us say something closer to, "unload an item until there aren't any left."

Here's what that second type of loop might look like in Ruby: 

for question in @questionList
  question = question.chomp
  f.puts question
end

Again, don't sweat it if you don't get it.

Conditionals

There's one more thing we might might need if we're going to unload the dishwasher efficiently and correctly. We have to be able to decide where to put each item, right? When we do it we might think to ourselves, "If I grab a bowl I'll put it in the bowl cupboard." Programmers do this with something called, "Conditionals."

Probably the most common conditional in Ruby and most other languages is the If statement. Here's an example: 

if name == "Frank"
  puts "Frank's not here"
end

One thing to notice is the "==". That's not a typo. When we assign values to a variable we use "=" but if we want to check if two things are equal we use "==".

You might have guessed from the name that @questionList is an Array. In this case an array of Strings. That's right, the items that make up an Array's list are Instances of various Data Types.
